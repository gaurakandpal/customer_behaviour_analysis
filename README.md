Retail Customer Behaviour & Shopping Trends Analysis
**Project Overview**
This project demonstrates a complete end-to-end data analytics workflow, starting from raw retail customer data and progressing through data cleaning, SQL-based business analysis, and interactive dashboard development. The objective was to analyse customer shopping behaviour, 
identify purchasing patterns, evaluate product performance, and generate actionable business insights to support data-driven decision-making. 
The project follows an industry-standard analytics pipeline commonly used by data analysts and business intelligence professionals.

**Objectives**

* Clean and prepare raw retail customer data for analysis.
* Perform exploratory data analysis (EDA) using Python.
* Store and query the cleaned dataset using PostgreSQL.
* Analyse customer demographics, shopping behaviour, and purchasing trends.
* Develop an interactive Power BI dashboard for business stakeholders.
* Generate actionable recommendations based on analytical findings.

**Tech Stack**
 Tool                          Purpose                                      |

| **Python (Pandas, NumPy)** | Data cleaning and preprocessing              |
| **PostgreSQL**             | Data storage and business analysis using SQL |
| **Power BI**               | Dashboard development and data visualization |
| **Power Query**            | Data transformation within Power BI          |
| **DAX**                    | KPI calculations and dashboard measures      |

Project Workflow

**Data Cleaning & Preparation (Python)** 

The raw dataset was cleaned using Python to improve data quality before analysis.
* Handling missing values
* Removing duplicate records
* Correcting inconsistent values
* Standardising categorical variables
* Validating data types
* Preparing the dataset for SQL analysis
* Exporting the cleaned dataset for PostgreSQL

**SQL Business Analysis (PostgreSQL)**
The cleaned dataset was imported into PostgreSQL where SQL queries were written to answer business questions.

* Customer demographics
* Revenue by age group and gender
* Product category performance
* Best-selling products
* Seasonal purchasing trends
* Discount utilisation
* Customer segmentation
* Customer review ratings
* Shipping preferences
* Payment method analysis
* Purchase frequency analysis
* Revenue contribution across customer groups

**Advanced SQL concepts used:**
* Aggregate Functions
* CASE Statements
* GROUP BY
* HAVING
* ORDER BY
* Common Table Expressions (CTEs)
* Window Functions
* Conditional Aggregation

**Dashboard Development (Power BI)**
The SQL dataset was connected to Power BI to create an interactive business dashboard.
Dashboard features include:
* KPI Cards
* Revenue Analysis
* Customer Demographics
* Product Performance
* Discount Analysis
* Customer Satisfaction
* Seasonal Shopping Trends
* Payment Method Analysis
* Shipping Type Analysis
* Interactive Filters & Slicers

**Key Business Insights**
* Young adults generated the highest overall revenue.
* Some products achieved high sales despite relatively lower customer ratings, indicating opportunities for quality improvements.
* Discounts significantly influenced purchasing behaviour across several product categories.
* Seasonal trends revealed variations in customer demand, supporting inventory and promotional planning.
* Returning and loyal customers contributed a substantial share of overall purchases, highlighting the importance of customer retention strategies.
